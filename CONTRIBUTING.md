# Contributing

Thanks for your interest in contributing.

## Basic rules

- Follow the code style guidelines in `docs/`.
- Use the Git conventions described in `docs/`.
- Open a Pull Request (PR) for changes.

## Branching

- `main`: production
- `develop`: development
- `feature/*`: new features
- `fix/*`: bug fixes

## Commit messages

Use clear commit prefixes:

- `feat:`
- `fix:`
- `docs:`
- `refactor:`

## Pull Requests

- Describe what changed and why.
- Add screenshots or examples when relevant.
- Ensure no secrets are committed.
